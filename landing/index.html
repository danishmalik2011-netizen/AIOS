FAQ

Questions, answered

Everything you need to know before you hit download.

Which models and providers does AIOS support?

AIOS is provider-agnostic. Out of the box it connects to Anthropic (Claude), OpenAI (GPT-4o, o-series), Google (Gemini), and Ollama for fully local models. It also speaks the OpenAI-compatible protocol, so any endpoint that exposes a /v1/models and Chat Completions API works — OpenRouter, Groq, Together, Mistral, DeepSeek, Azure OpenAI, LM Studio, and your own self-hosted gateways. When you save a key (or register a custom base URL), AIOS auto-fetches the provider's live model catalogue and adds every model to your system — no manual copy-paste of model IDs.

Is AIOS really local-first and private?

Yes. The desktop app runs entirely on your machine — no mandatory account, no telemetry, no analytics beacons. Your source code, conversation history, and agent plans are cached locally under ~/.aios/. API keys are stored in your OS credential vault (Windows Credential Manager), never in plaintext. Data only leaves your device when you explicitly call a cloud model or connect an external MCP tool server.

How does the multi-agent "Fleet" actually work?

A planner decomposes your goal into ordered steps, then delegates each to specialized built-in agents — Architect (design), CodeSmith (implementation), Sentinel (review/security), TestRunner (verification), and DeployBot (ship). Each agent can be bound to its own provider and model, so you might run Architect on Claude while TestRunner uses a fast local SLM. Agents share a working directory and an unbounded context window, and every file mutation is gated behind a plan you approve first.

Do I need Docker or DevOps experience to use it?

No. Running aios "scaffold docker workspace" generates the full environment, Docker config, and an agent team for you. Power users can export and version the resulting aios.yaml. The same tasks also run directly on your host shell if you prefer not to containerize.

What can the agents actually do on my machine?

Agents ship with 12 core system tools: read, write, and surgical patch files; directory listing; regex code search; command/shell execution (with a sandboxed PTY); git operations; and native tool-calling. Through MCP they can also drive databases, browsers, and third-party APIs. Every command is shown to you before execution, and you can deny, edit, or approve it.

Is it free, and what do I pay for?

The Windows (and macOS/Linux) desktop app is a free, code-signed installer with automatic in-app updates. AIOS itself costs nothing — you only pay the token usage charged by whichever model provider(s) you connect. Running Ollama or other local models is completely free with no external API cost.

Which platforms are supported?

AIOS ships a native Windows desktop build (NSIS installer) with auto-update, plus macOS (.dmg) and Linux (.AppImage) targets. There is also a cross-platform CLI (npm i -g aios) that runs the same agent engine in your terminal, and you can build from source on any platform with Node.js 20+.

How do updates work?

The desktop app checks a signed latest.yml manifest on launch and downloads differential blockmap updates in the background — no reinstall needed. The CLI updates via npm update -g aios. Release notes for every version are published on the site and in the app's update viewer.

Can I use AIOS for a team or commercial project?

Yes. AIOS is built for real software creation — scaffolding apps, generating UI/UX, running test suites, and deploying. Your code and configs remain yours; nothing is uploaded to a vendor. For shared agent rosters or CI integration, export your aios.yaml and commit it alongside the project.
