WHAT'S INSIDE

Everything the agents need to build.

A complete toolkit that lets AI work on real codebases — safely, and with you in charge.

Multi-Agent Fleet Director

Automated workflow decomposition. Tasks are delegated to specialized built-in agents: Architect, CodeSmith, Sentinel, TestRunner, and DeployBot.

Model Context Protocol (MCP)

Full integration with external MCP servers. Cache their schemas and seamlessly execute tools alongside AIOS's built-in toolkit.

BYOK Architecture

Native Tool Calling

Plan-Before-Act & Verify

Clean Git History

What's new

Shipped recently

Continuous improvements to the agent operating system — here's what landed across the last few releases.

1.3.3

Minimalist Branding & Polish Overhaul

  • Signature Geometric Logo: Standardized the new brutalist geometric logo across all channels (desktop client, CLI banner, favicons).
  • CLI Diagnostic Startup: Added real hardware scanning (CPU/RAM detection) and diagnostic logs on startup to mimic rig.ai styling.
  • Design Uniformity: Simplified primary gradients in the default Slate theme to flat, minimalistic accent colors to match the landing page.
  • No-Fake Guarantee: Cleaned up CLI subcommands and workspaces mockup code inputs to reflect real execution patterns.
1.3.2

Interactive Code Canvas, Visual Fleet & Mini-Browser

  • Interactive Canvas: Clickable diff badges now instantly open proposed file changes side-by-side in the Code Canvas. Heads-up diff reviews are pre-calculated for all modification tools.
  • Visual Fleet Pipelines: Headless fleet executions are now mapped directly to the visual canvas and interactive tool executors, allowing live code updates while agent roster tasks run.
  • Mini-Browser Live Preview: Added multi-tab support, back/forward controls, enter-to-navigate URL fields, and native project ZIP exports. Resolved fullscreen toggling with customizable device-width hover corner overlays.
  • Reliable Auto-Updates: Fixed installer relaunch conflicts, bringing back the interactive NSIS installer wizard to prevent app crashes on update.
  • Dynamic Secret Key Hydration: Added asynchronous fallbacks and store subscription listeners to automatically sync custom keys (Gemini, Groq, OpenRouter) and prevent voice-transcription key resolution race conditions.
1.2.8

Incremental Writes, System Attachments & clean UI

  • Incremental building. Implemented patch_file and append_file tool calls in the desktop app. Enables agents to build large files step-by-step instead of doing huge rewrites, eliminating completion truncation issues.
  • Any-file attachments. Added support for attaching any local system file from the native chooser dialog (not just workspace files). Fully reads external contents into the agent's context.
  • Voice and UX Polish. Relocated the Speech Read Aloud button underneath specific agent responses with faster playback rate. Wrapped speech calls in handlers to prevent renderer freezes. Removed the top header bar from the fleet chat canvas for a cleaner, spacious editor view.
1.2.7

AI Speech Controls, Prompt Polish & UI UX Polish

  • AI Prompt Polish. Added a Sparkles prompt optimization button next to the composer inputs. Uses the background LLM to automatically fix speech-to-text grammar, repair typos, and restate prompts cleanly.
  • Speech-to-Text & Read Aloud. Implemented Speech Recognition on the microphone button (flashes red when active) and Text-to-Speech synthesis on the audio waveform button (pulses blue when active) to read responses.
  • UX Refinements. Cleaned the Left Rail sidebar by removing duplicate tabs, removing the profile footer, and adding dynamic collapse layouts (hiding search, icon-only new chat/updater). Added dismiss 'X' to context limit warning. Fixed terminal scroll wrap.
1.2.6

Premium Claude-like UI, PDF Cards & Context limits

  • Claude-like reskin. Entirely redesigned Left Rail with segmented Navigation Tabs, New Chat button, flat Pinned/Recents global sections, Relaunch Check card, and Danish Profile card.
  • Interactive PDF widgets. Attached PDF documents are parsed and displayed as beautiful document cards with a native 'Download and open' button running on the OS viewer.
  • Pill input composer. Reshaped prompt textarea with integrated Model Pill selector, mic/waveform speech controls, and a dedicated active attachment row.
1.2.5

Reliability, regenerate & theme fixes

  • HTTP 400 hardened. Transient provider 400/429/5xx errors now auto-retry with exponential backoff, and streamed tool-call arguments are sanitized — the recurring "Provider error 400" after a long run is gone.
  • Regenerate any message. Every assistant reply now has a Regenerate (and Retry-on-error) action so you can re-run a turn without losing the thread.
  • Light theme perfected. Removed hardcoded dark colors so hover, padding, and borders follow the active theme in both light and dark modes — no more dark bleed in light.
1.2.3

MCP tool servers, smarter fleet & reliability

  • MCP tool servers. Connect external Model Context Protocol servers in Settings ? MCP Servers so your agents gain real-world tools (browsers, databases, deploy hooks) — no recompile, zero new dependencies.
  • Fleet respects your roster. "Run with Fleet" now uses each agent's configured provider/model from the Agent Roster (no silent local fallback) and can actually call tools during background runs.
  • Focused multi-agent runs. CrewAI-style role backstories + contracts and LangGraph-style handoffs keep specialists in their lane; upstream context is now structured and summarized.
  • Recovery & transparency. Truncated/empty model replies auto-compact history and retry instead of dead-ending; a new Release Notes section in Settings summarizes every recent change.
1.1.20

Fleet Director & editable Agent Roster

  • Run with Fleet. A toggle in the Agents composer. The background Director decomposes your goal and routes each step to the right specialist — Architect plans, CodeSmith builds, Sentinel reviews, TestRunner tests, DeployBot ships.
  • Shared + prior-agent context. Every agent in a run receives the same project brief and memory, plus the output of the agents before it, so the fleet builds on each other's work instead of working in silos.
  • Offline mode & BYOK. Native routing is deterministic and runs offline. The core AI loop is powered by Bring Your Own Key (BYOK) architecture, giving you total control over the APIs you call while ensuring your code never touches our servers.
  • Editable Agent Roster. In Account ? Agent Roster, set each agent's provider and model from the same picker as the composer. Choices persist across updates.

FAQ

Questions, answered

Everything you need to know before you hit download.

Is AIOS really local-first and private?

Yes. AIOS runs entirely on your machine — no account, no telemetry, no data leaves your device unless you explicitly connect an external tool server. Secrets and keys stay local.

Which models can I use?

Claude, GPT-4, and local models — routed per task. Each agent in your roster can have its own provider and model, and Fleet respects those choices on every run.

Do I need to know Docker?

No. One command (aios deploy --preset claude) scaffolds the full environment, Docker config, and agent team for you. Power users can export and version the aios.yaml.

Is it free to download?

The Windows desktop app is a free, signed installer with automatic updates. You only pay for the model provider tokens you choose to use.
