Core capabilities

Built for how AI actually builds software

Every surface is agent-addressable. Every action is reviewable. Every decision is traceable.

Multi-model orchestration

Claude, GPT-4, local models — routed automatically by task. No API keys in code. Agent-specific model selection per step.

Visual workflow engine

Drag-and-drop DAGs for agent pipelines. Parallel execution, conditional branching, human-in-the-loop gates, rollback points.

Persistent project memory

Vector-indexed codebase, decisions, and docs. Agents retrieve context instantly — no re-reading files every turn.

Native terminal integration

Isolated Docker terminals per agent. Real-time stdout/stderr streaming. Shell, git, test runners — all agent-controllable.

Security by default

Local-first, zero telemetry. Secrets never leave your machine. Sandboxed execution with capability-based permissions.

Git-native collaboration

Every agent action is a commit. PRs auto-generated with summaries. Review, merge, revert — standard git workflow.

Workspaces

Isolated environments, one command

Spin up project-scoped workspaces with pre-configured agents, tools, and context. Share via reproducible config.

Preset templates

Web, CLI, API, ML, mobile — one flag scaffolds the full agent team, docker config, and task queue.

Shared context layer

Workspaces share a project memory index. Agents across sessions pick up where others left off.

Portable configuration

Export workspace as agy.yaml — version it, share it, CI it. Reproduce environments identically.

What's new

Shipped recently

Continuous improvements to the agent operating system — here's what landed across the last few releases.

1.2.4

Reliability, regenerate & theme fixes

  • HTTP 400 hardened. Transient provider 400/429/5xx errors now auto-retry with exponential backoff, and streamed tool-call arguments are sanitized — the recurring "Provider error 400" after a long run is gone.
  • Regenerate any message. Every assistant reply now has a Regenerate (and Retry-on-error) action so you can re-run a turn without losing the thread.
  • Light theme perfected. Removed hardcoded dark colors so hover, padding, and borders follow the active theme in both light and dark modes — no more dark bleed in light.
1.2.3

MCP tool servers, smarter fleet & reliability

  • MCP tool servers. Connect external Model Context Protocol servers in Settings → MCP Servers so your agents gain real-world tools (browsers, databases, deploy hooks) — no recompile, zero new dependencies.
  • Fleet respects your roster. "Run with Fleet" now uses each agent's configured provider/model from the Agent Roster (no silent local fallback) and can actually call tools during background runs.
  • Focused multi-agent runs. CrewAI-style role backstories + contracts and LangGraph-style handoffs keep specialists in their lane; upstream context is now structured and summarized.
  • Recovery & transparency. Truncated/empty model replies auto-compact history and retry instead of dead-ending; a new Release Notes section in Settings summarizes every recent change.
1.1.20

Fleet Director & editable Agent Roster

  • Run with Fleet. A toggle in the Agents composer. The background Director decomposes your goal and routes each step to the right specialist — Architect plans, CodeSmith builds, Sentinel reviews, TestRunner tests, DeployBot ships.
  • Shared + prior-agent context. Every agent in a run receives the same project brief and memory, plus the output of the agents before it, so the fleet builds on each other's work instead of working in silos.
  • Offline by default. Routing is a deterministic intent classifier — no API key required. An optional LLM planning pass uses your active session's model when a key is present.
  • Editable Agent Roster. In Account → Agent Roster, set each agent's provider and model from the same picker as the composer. Choices persist across updates.
1.1.19

Agent transparency

  • Live "Thinking…" timer and a collapsible Work Summary of every tool step an agent took.
  • Composer context bar showing token usage, so you always know how much context a turn consumed.
1.1.18

Workflow orchestration & canvas

  • Visual DAG workflow engine with parallel execution and human-in-the-loop gates.
  • Agent Canvas (live plans & artifacts), Follow Agent, and Auto Access (trust mode).
  • Multi-model orchestration with per-agent provider/model selection.
