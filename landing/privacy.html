01 Our philosophy

AIOS is built on a simple premise: your code, your data, your machine. We don't believe in telemetry, mandatory accounts, or software that reports back to a server. The application runs entirely on your hardware — what happens in AIOS stays in AIOS, unless you explicitly choose to share it.

02 The short version

Local-firstEverything persistent lives in a folder you own.
No telemetryZero usage analytics or third-party trackers.
Keys stay putAPI keys are encrypted with Windows DPAPI.
You own itDelete the folder and the app is gone. No residue.

03 Data we collect

Very little — and only when you opt in. The desktop app collects nothing by default.

Application crash reports (opt-in)

If you enable crash reporting in Settings → Privacy, anonymous stack traces are sent to our error-tracking service (Sentry) to help us fix bugs. No code, file paths, or project content is included. You can disable this at any time.

Update checks

The app periodically contacts releases.aios.app to check for new versions. This request includes only the current version string and platform (Windows). No identifiers, no hardware fingerprints.

License validation (Pro only)

If you purchase a Pro license, a signed license token is validated offline. No recurring license checks. No account required.

04 Data we don't collect

  • No telemetry. No usage analytics, no feature-adoption tracking, no heatmaps.
  • No code scanning. Your source code never leaves your machine unless you invoke an agent that calls an external LLM API.
  • No file-index uploads. The vector index for project memory is built and stored locally in ~/.aios/memory/.
  • No session recording. Terminal output, agent chats, and workflow runs stay on disk.
  • No account system. You don't sign in. There's no profile, no email capture, no password.
  • No third-party trackers. No Google Analytics, Mixpanel, Amplitude, Segment, or similar.

05 Local-first architecture

AIOS stores all persistent data in a single, user-owned directory:

%USERPROFILE%\.aios\
├── memory/          # Vector embeddings (local SQLite + FAISS)
├── workspaces/      # Workspace configs (agy.yaml)
├── logs/            # Local agent/terminal logs (rotated, 7 days)
├── cache/           # Model cache, downloaded assets
└── settings.json    # User preferences (theme, keys, etc.)

You can delete ~/.aios/ at any time to fully reset the application. No registry keys, no hidden AppData folders, no system-level services.

Project memory

When you enable Project Memory for a workspace, AIOS creates a local vector index of your codebase using sentence-transformers (runs on your CPU/GPU). Embeddings never leave your machine. The index is used only to answer agent queries like “find the auth middleware” or “show recent changes to payments.”

06 Agent execution & API keys

AIOS agents can call external LLM providers (Anthropic, OpenAI, Google, Ollama, custom endpoints). Here's how it works:

API key storage

Keys are stored in Windows DPAPI (Data Protection API) — encrypted with your user credentials. They are never written to disk in plaintext, never synced, never logged. The UI displays only a masked preview (e.g., sk-ant-•••••••••).

Request flow

  1. Agent prepares a prompt with local context (retrieved from your project memory).
  2. Request is sent directly from your machine to the provider's API endpoint (HTTPS, TLS 1.3).
  3. Response streams back to your machine. AIOS does not proxy, log, or inspect the content.

We never see your prompts or completions.

Local models (Ollama, LM Studio, etc.)

When configured to use a local inference server, all traffic stays on localhost or your LAN. Zero external network calls.

07 Auto-updates

AIOS checks for updates on startup and every 24 hours thereafter via a simple HTTPS GET to https://releases.aios.app/latest.yml. The response contains version metadata and a signed installer URL. The request includes only:

  • Current version (e.g., 1.1.10)
  • Platform (win32)
  • Architecture (x64)

No unique device IDs, no user IDs, no IP logging beyond standard CDN logs (which we don't access). You can disable automatic checks in Settings → Updates.

08 Your rights

  • Access & portability: All your data lives in ~/.aios/. Copy it, back it up, move it — it's yours.
  • Deletion: Delete ~/.aios/ and uninstall the app. Zero residue.
  • No “right to be forgotten” requests needed: We don't have your data to forget.
  • Opt-out: Crash reporting and update checks are toggleable in Settings → Privacy.
