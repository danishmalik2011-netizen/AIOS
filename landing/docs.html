Introduction

AIOS (AI Agent Operating System) is a complete, local-first AI software development environment. Unlike other AI agents that upload your source files to remote clouds, AIOS coordinates tool runs and compiles your code completely on your local machine.

Bring your own API keys (BYOK), enforce strict local execution sandboxes, and run multiple specialized agents in a background loop. AIOS runs with zero telemetry—your files, prompts, and tokens never leave your storage.

Installation

To initialize the global CLI, run the NPM package link or fetch the desktop package installer directly:

$ npm install -g aios

Verify that your configuration environment works by querying the system version:

$ aios --version
aios v1.3.4

Windows users can also download the signed desktop installer from the downloads section on the landing page, which houses the desktop canvas and active directory configuration panels.

BYOK Credentials & Settings

Your workspace preferences, agent presets, and API keys are stored securely inside your user directory:

~/.aios/config.json

On initial launch, AIOS prompts you to enter your credentials. You can also configure them manually by editing the config JSON or running the /provider command inside the REPL loop.

Active session history state metrics are cached inside ~/.aios/sessions/<hash>.json, where the session ID matches the SHA-256 hash of your active project root folder.

CLI Flags & Syntax

AIOS accepts various command-line options to modify execution flow. Pass prompt strings directly for single-turn modifications, or launch without parameters to open the REPL shell.

Flag Description Syntax Example
-y, --yes Bypasses manual tool execution prompts. Grant auto-approvals. aios -y "fix type errors"
-r, --root Manually set the target workspace project root directory. aios -r ./packages/server
-m, --model Specify the LLM model target override for this execution. aios -m claude-3-5-sonnet
-h, --help Lists all command arguments and global system flags. aios --help

Agent Fleet Roles

AIOS splits software creation tasks among multiple specialized agent roles, orchestrated by a central coordinator:

Director Coordinates the main loop, maps workspace file trees, and delegates checklist tasks.
Architect Researches codebase syntax, creates checklists, and plans file modifications.
CodeSmith Executes precise file edits, appends incremental lines, and writes code.
Sentinel Validates build flags, reviews compiler logs, and checks git diff states.
TestRunner Runs localized test suites to ensure changes do not break existing regressions.
DeployBot Automates build caching and updates installer bundles for release updates.

REPL Slash Commands

When running in interactive REPL mode, you can control the environment state with the following slash commands:

Command Description
/model <id> Swap the current execution LLM model model immediately.
/provider Configure model provider endpoint URLs and local API keys.
/status Queries active agent fleet states and mapped directories.
/tokens Displays accumulated prompt, output, and cache token statistics.
/clear Wipes the interactive screen console logs.
/exit Saves local session history states and exits the shell.
